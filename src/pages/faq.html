즉시 해결

터미널이 어디 있나요? 찾을 수가 없어요!

Mac에서 터미널을 찾는 가장 쉬운 방법을 알려드릴게요:

  1. Command (Cmd) ⌘ + Space 를 누르세요 (Spotlight 검색)
  2. "터미널" 또는 "Terminal"이라고 입력하세요
  3. Enter 를 누르면 터미널이 열려요!
터미널을 열면 이렇게 보입니다
Last login: Mon Jan 8 10:32:15 on ttys001 사용자명@MacBook-Pro ~ %
💡 Tip

자주 사용하실 거라면 터미널을 Dock에 추가해두세요. 터미널 아이콘을 우클릭하고 "옵션 > Dock에 유지"를 선택하면 됩니다.

비밀번호를 입력해도 아무것도 안 보여요!

이건 정상이에요! 터미널은 보안상 비밀번호를 입력할 때 아무것도 표시하지 않습니다.

비밀번호 입력 시 화면 - 정상적인 상황
사용자명@MacBook-Pro ~ % sudo npm install -g claude Password: # ← 비밀번호를 입력해도 화면에 아무것도 표시되지 않습니다 (보안상 정상)
  1. 그냥 Mac 로그인 비밀번호를 입력하세요
  2. 화면에 아무것도 안 보여도 입력은 되고 있어요
  3. 다 입력했으면 Enter 를 누르세요
비밀번호가 틀렸다고 나오면 다시 천천히 입력해보세요. Caps Lock이 켜져있는지도 확인하세요!

Permission denied 오류가 계속 나와요

권한 문제예요. Mac에서는 시스템 폴더에 뭔가를 설치할 때 관리자 권한이 필요해요.

잘못된 방법 - 권한 오류 발생
사용자명@MacBook-Pro ~ % npm install -g claude npm ERR! Error: EACCES: permission denied, mkdir '/usr/local/lib/node_modules/claude' npm ERR! The operation was rejected by your operating system. npm ERR! It is likely you do not have the permissions to access this file as the current user
올바른 방법 - sudo 사용
사용자명@MacBook-Pro ~ % sudo npm install -g claude Password: # ← 여기서 Mac 비밀번호를 입력하면 설치가 진행됩니다

sudo를 앞에 붙이면 관리자 권한으로 실행됩니다.

명령 프롬프트가 어디 있나요?

Windows에서 명령 프롬프트를 여는 가장 쉬운 방법들:

  1. 방법 1: Windows (Win) ⊞ + R 을 누르고 "cmd" 입력 후 Enter
  2. 방법 2: 시작 버튼 클릭 → "cmd" 또는 "명령 프롬프트" 검색
  3. 방법 3: Windows (Win) ⊞ + X 누르고 "명령 프롬프트" 선택
💡 팁

명령 프롬프트를 자주 사용하신다면 작업 표시줄에 고정해두면 편해요.

액세스가 거부되었습니다 오류가 나와요

Windows에서는 관리자 권한으로 명령 프롬프트를 실행해야 할 때가 있어요.

  1. 명령 프롬프트를 마우스 우클릭
  2. "관리자 권한으로 실행" 선택
  3. 사용자 계정 컨트롤 창이 뜨면 "예" 클릭
  4. 다시 명령어를 입력해보세요

불안감 해소

설치가 너무 오래 걸려요 (5분 넘게 진행 중)

프로그램 설치는 여러 파일을 다운로드하고 설정하는 과정이라 시간이 걸려요. 특히 처음 설치할 때는 더 그래요.

설치 진행 중 화면 - 시간이 오래 걸리는 상황
Downloading packages... Installing dependencies... Installing: express@4.18.2 Installing: lodash@4.17.21 Installing: moment@2.29.4 # ← 이런 식으로 여러 프로그램을 차례대로 설치하고 있어요!
✅ 정상이에요!

인터넷 속도에 따라 5-10분 정도 걸릴 수 있어요. 커피 한 잔 하고 오세요!

설치 중에 WARN 메시지가 여러 개 나왔어요

npm은 설치 중에 다양한 경고를 보여주지만, 대부분은 무시해도 됩니다.

설치 중 경고 메시지 - 정상적인 상황
사용자명@MacBook-Pro ~ % npm install some-package
C:\Users\사용자명> npm install some-package # → 명령 프롬프트(cmd) 기준입니다
npm WARN deprecated package@1.0.0: This package is deprecated # → 구버전 패키지라는 경고 (정상) npm WARN optional SKIPPING OPTIONAL DEPENDENCY # → 선택적 구성요소 건너뜀 (정상) npm WARN some-module@2.1.0 requires a peer dependency # → 추가 패키지가 필요하다는 알림 (정상)
💡 알아두세요

WARN은 경고일 뿐이에요. ERROR가 아니라면 정상적으로 설치되고 있는 겁니다!

터미널에 이상한 기호나 색깔이 나와요

터미널은 다양한 색상과 기호로 정보를 구분해서 보여줍니다.

터미널 색상과 기호 예시
사용자명@MacBook-Pro ~ % npm install express
C:\Users\사용자명> npm install express
added 64 packages in 3s # → 기본 색상 = 성공/완료 npm WARN deprecated package@1.0.0: This package is deprecated # → 노란색 WARN = 경고 (무시 가능) npm ERR! code ENOENT npm ERR! syscall open
npm ERR! path /Users/username/package.json
npm ERR! path C:\Users\사용자명\package.json
# → 빨간색 ERR! = 오류 (확인 필요)
사용자명@MacBook-Pro ~ % # → % 또는 $ = 명령어 입력 대기 중
C:\Users\사용자명> # → > = 명령어 입력 대기 중

기초 지식

복사/붙여넣기는 어떻게 하나요?

터미널에서는 복사/붙여넣기 방법이 조금 달라요:

  • 복사: 텍스트 선택 후 Command (Cmd) ⌘ + C
  • 붙여넣기: Command (Cmd) ⌘ + V
  • 복사: 텍스트 선택 후 Control (Ctrl) + C 또는 마우스 우클릭
  • 붙여넣기: Control (Ctrl) + V 또는 마우스 우클릭

터미널에서 실수로 이상한 걸 쳤어요

괜찮아요! 터미널에서 빠져나오는 방법들:

  • Control (Ctrl) + C : 현재 실행 중인 것을 중단
  • Control (Ctrl) + D : 프로그램 종료
  • exit 입력 : 터미널 종료
  • 최후의 수단: 터미널 창을 그냥 닫고 새로 열기

cd, ls 같은 명령어는 뭔가요?

터미널에서 자주 쓰는 기본 명령어들이에요:

pwd - 현재 위치 확인하기
사용자명@MacBook-Pro ~ % pwd /Users/yourname # → 지금 /Users/yourname 폴더에 있다는 뜻이에요
cd - 현재 위치 확인하기
C:\Users\사용자명> cd C:\Users\사용자명 # → 지금 C:\Users\사용자명 폴더에 있다는 뜻이에요
ls - 파일과 폴더 목록 보기
사용자명@MacBook-Pro ~ % ls Desktop Documents Downloads Pictures # → 현재 폴더에 있는 파일과 폴더들이 보여요
dir - 파일과 폴더 목록 보기
C:\Users\사용자명> dir 2025-07-16 오전 10:30 . 2025-07-16 오전 10:30 .. 2025-07-16 오전 09:15 Desktop 2025-07-15 오후 02:30 Documents 2025-07-14 오후 05:45 Downloads 2025-07-13 오전 11:20 Pictures # → 현재 폴더에 있는 파일과 폴더들이 보여요
cd - 다른 폴더로 이동하기
사용자명@MacBook-Pro ~ % cd Desktop
사용자명@MacBook-Pro Desktop %
# → Desktop 폴더로 이동했어요! (프롬프트가 바뀐 걸 보세요)
cd - 다른 폴더로 이동하기
C:\Users\사용자명> cd Desktop
C:\Users\사용자명\Desktop>
# → Desktop 폴더로 이동했어요! (프롬프트가 바뀐 걸 보세요)
cd .. - 상위 폴더로 돌아가기
사용자명@MacBook-Pro Desktop % cd ..
사용자명@MacBook-Pro ~ %
# → Desktop에서 홈 폴더로 돌아왔어요
cd .. - 상위 폴더로 돌아가기
C:\Users\사용자명\Desktop> cd ..
C:\Users\사용자명>
# → Desktop에서 사용자 폴더로 돌아왔어요
mkdir - 새 폴더 만들기
사용자명@MacBook-Pro ~ % mkdir my-project
사용자명@MacBook-Pro ~ % ls
Desktop Documents Downloads Pictures my-project # → my-project 폴더가 새로 만들어졌어요!
mkdir - 새 폴더 만들기
C:\Users\사용자명> mkdir my-project
C:\Users\사용자명> dir
2025-07-16 오전 10:35 my-project 2025-07-16 오전 10:30 Desktop 2025-07-15 오후 02:30 Documents # → my-project 폴더가 새로 만들어졌어요!
clear - 화면 정리하기
사용자명@MacBook-Pro ~ % clear # → 터미널 화면이 깨끗해져요! (이전 내용은 스크롤로 볼 수 있어요)
cls - 화면 정리하기
C:\Users\사용자명> cls # → 터미널 화면이 깨끗해져요!
cat - 파일 내용 보기
사용자명@MacBook-Pro ~ % cat package.json { "name": "my-project", "version": "1.0.0", "description": "My first project" } # → 파일의 전체 내용이 보여요
type - 파일 내용 보기
C:\Users\사용자명> type package.json { "name": "my-project", "version": "1.0.0", "description": "My first project" } # → 파일의 전체 내용이 보여요
which - 프로그램 설치 위치 확인
사용자명@MacBook-Pro ~ % which node
/usr/local/bin/node
사용자명@MacBook-Pro ~ % which npm
/usr/local/bin/npm # → Node.js와 npm이 제대로 설치되어 있네요!
where - 프로그램 설치 위치 확인
C:\Users\사용자명> where node
C:\Program Files\nodejs\node.exe
C:\Users\사용자명> where npm
C:\Program Files\nodejs\npm.cmd # → Node.js와 npm이 제대로 설치되어 있네요!
cp - 파일 복사하기
사용자명@MacBook-Pro ~ % cp README.md README-backup.md
사용자명@MacBook-Pro ~ % ls *.md
README-backup.md README.md # → README.md 파일의 복사본이 만들어졌어요
copy - 파일 복사하기
C:\Users\사용자명> copy README.md README-backup.md
1개 파일이 복사되었습니다.
C:\Users\사용자명> dir *.md
2025-07-16 오전 10:40 1,245 README-backup.md 2025-07-16 오전 10:30 1,245 README.md # → README.md 파일의 복사본이 만들어졌어요
mv - 파일 이동하거나 이름 바꾸기
사용자명@MacBook-Pro ~ % mv old-name.txt new-name.txt # → 파일 이름이 바뀌었어요 사용자명@MacBook-Pro ~ % mv file.txt Desktop/ # → file.txt가 Desktop 폴더로 이동했어요
move 또는 ren - 파일 이동하거나 이름 바꾸기
C:\Users\사용자명> ren old-name.txt new-name.txt # → 파일 이름이 바뀌었어요 C:\Users\사용자명> move file.txt Desktop 1개 파일을 이동했습니다. # → file.txt가 Desktop 폴더로 이동했어요
touch - 빈 파일 만들기
사용자명@MacBook-Pro ~ % touch index.js
사용자명@MacBook-Pro ~ % ls *.js
index.js # → 빈 index.js 파일이 만들어졌어요
echo > - 빈 파일 만들기
C:\Users\사용자명> echo. > index.js
C:\Users\사용자명> dir *.js
2025-07-16 오전 10:45 2 index.js # → index.js 파일이 만들어졌어요
